## Summary

- Smart home devices (like Amazon Echo and Google Home) are designed for trusting households, which creates dangerous vulnerabilities in domestic abuse situations. In these scenarios, the abuser is typically the "primary user" (admin) and the survivor is the "secondary user". The findings point to security design flaw, whereby, the security largely favors the abuser. [[#Contributions (What?)]]

## Contributions (What?)

- **Asymmetric Visibility:** The system protects the primary user's privacy, meaning a secondary user (survivor) is completely blocked from seeing the primary user's activity or configurations.
- **Zero System Feedback:** Devices rarely notify users when settings are changed or monitoring is active[cite: 1]. Survivors must rely purely on their own memory to detect if they are being surveilled[cite: 1].
- **Interface Inconsistency:** Privacy and security settings are scattered and look completely different depending on whether you check the mobile app, web browser, or physical device[cite: 1]. 
- **High Friction for Escaping:** Attempting to untangle linked accounts or safely exit the shared network requires confusing, difficult context-switches across multiple platforms[cite: 1].
- **The Usability Paradox:** A device that operates silently and seamlessly is considered highly "usable" by tech companies[cite: 1]. However, this exact silent operation perfectly enables an abuser's hidden surveillance while actively preventing a survivor from protecting themselves[cite: 1].
